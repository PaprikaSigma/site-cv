Relațiile dintre clase în POO

Programare Orientată pe Obiecte · C#

Acasă / Relații POO

În programarea orientată pe obiecte, un sistem nu este doar o colecție de clase izolate.
Un program real este o rețea de relații între obiecte. Un programator profesionist petrece mai mult timp proiectând relațiile decât scriind codul.

Moștenire Agregare Compunere Dependență

1. Moștenire (Inheritance)

„este un tip de" IS-A Cuvânt cheie: :

Moștenirea permite unei clase derivate (child) să preia toate câmpurile, proprietățile și metodele clasei de bază (parent) și să le extindă sau suprascrie. Aceasta elimină duplicarea codului și creează o ierarhie logică între entități.

  • Clasa derivată moștenește toți membrii non-privați ai clasei de bază.
  • Se poate suprascrie comportamentul cu override (în combinație cu virtual).
  • C# suportă moștenire unică (o clasă poate extinde o singură clasă), dar interfețe multiple.
Animal
+ Nume : string
+ Respira()
moștenire ▲─────
Caine
+ Rasa : string
+ Latra()
moștenire ▲─────
CainePolitist
+ NumarInsigna : int
+ Patruleaza()
C# 
// Clasa de bază
public class Animal
{
    public string Nume { get; set; }

    public virtual void FaSunet()
    {
        Console.WriteLine($"{Nume} scoate un sunet.");
    }
}

// Clasa derivată — „Câinele ESTE UN Animal"
public class Caine : Animal
{
    public string Rasa { get; set; }

    public override void FaSunet()  // suprascriere
    {
        Console.WriteLine($"{Nume} latră: Ham ham!");
    }

    public void Latra() => Console.WriteLine("Ham! Ham!");
}

// Ierarhie pe mai multe niveluri
public class CainePolitist : Caine
{
    public int NumarInsigna { get; set; }

    public void Patruleaza()
    {
        Console.WriteLine($"Câinele #{NumarInsigna} patrulează zona.");
    }
}

// Utilizare
CainePolitist rex = new CainePolitist { Nume = "Rex", Rasa = "Ciobanesc", NumarInsigna = 42 };
rex.FaSunet();    // Rex latră: Ham ham!
rex.Patruleaza(); // Câinele #42 patrulează zona.

2. Agregare (Aggregation)

„are un" HAS-A (slab) Obiectul parte poate exista independent

Agregarea este o relație de tip „are un" în care un obiect conține o referință către alt obiect, dar cele două au cicluri de viață independente. Dacă containerul este distrus, obiectul conținut poate supraviețui.

  • Implementată de obicei ca o proprietate / câmp de tip referință.
  • Obiectul parte este creat în afara containerului și transmis prin constructor sau setter.
  • Exemplu clasic: un Profesor predă la o Scoala — dacă școala se închide, profesorul există în continuare.
Scoala
+ Profesori : List<Profesor>
◇──── agregare
Profesor
+ Nume : string
+ Materie : string
C# 
public class Profesor
{
    public string Nume    { get; set; }
    public string Materie { get; set; }

    public Profesor(string nume, string materie)
    {
        Nume    = nume;
        Materie = materie;
    }

    public void Preda() => Console.WriteLine($"{Nume} predă {Materie}.");
}

public class Scoala
{
    public string Denumire { get; set; }
    // Agregare — Profesorii sunt creați în afara Școlii
    private List<Profesor> _profesori = new();

    public void AdaugaProfesor(Profesor p) => _profesori.Add(p);

    public void AfiseazaPersonal()
    {
        foreach (var p in _profesori) p.Preda();
    }
}

// Profesorul există independent de școală
Profesor ionescu = new Profesor("Ionescu", "Matematică");
Scoala   ceef    = new Scoala   { Denumire = "CEEF" };

ceef.AdaugaProfesor(ionescu);
ceef.AfiseazaPersonal();

// Dacă ceef ar fi șters, ionescu rămâne valid

3. Compunere (Composition)

„este format din" HAS-A (puternic) Cicluri de viață legate

Compunerea este o formă mai puternică de agregare: obiectul parte nu poate exista fără containerul său. Dacă containerul este distrus, toate componentele sale sunt distruse împreună cu el. Obiectele parte sunt create în interiorul constructorului containerului.

  • Obiectul parte este instanțiat în interiorul clasei container.
  • Containerul deține responsabilitatea completă a duratei de viață.
  • Exemplu: o Masina este formată din Motor — dacă mașina e demolată, motorul nu mai are sens singur.
Masina
- motor : Motor
- cutieViteze : CutieViteze
◆──── compunere
Motor
+ Putere : int
+ Porneste()
CutieViteze
+ NrViteze : int
C# 
public class Motor
{
    public int PutereCP { get; }

    public Motor(int putere) => PutereCP = putere;

    public void Porneste() => Console.WriteLine($"Motor de {PutereCP} CP pornit! Vrrr...");
}

public class CutieViteze
{
    public int NrViteze { get; }
    public CutieViteze(int nr) => NrViteze = nr;
}

public class Masina
{
    public string Marca { get; }
    // Compunere — componentele sunt create în constructor
    private readonly Motor       _motor;
    private readonly CutieViteze _cutie;

    public Masina(string marca, int putere, int viteze)
    {
        Marca  = marca;
        _motor = new Motor(putere);       // creat INTERN
        _cutie = new CutieViteze(viteze);  // creat INTERN
    }

    public void Porneste()
    {
        Console.WriteLine($"=== {Marca} ===");
        _motor.Porneste();
        Console.WriteLine($"Cutie cu {_cutie.NrViteze} viteze angrenată.");
    }
}

Masina dacia = new Masina("Dacia Logan", 90, 5);
dacia.Porneste();
// === Dacia Logan ===
// Motor de 90 CP pornit! Vrrr...
// Cutie cu 5 viteze angrenată.

4. Dependență (Dependency)

„folosește temporar" USES-A Cel mai slab tip de relație

Dependența este cea mai slabă formă de relație dintre clase. O clasă folosește o altă clasă temporar — de obicei ca parametru al unei metode, variabilă locală sau valoare returnată — fără să o stocheze ca câmp.

  • Nu există câmp de tip referință — utilizarea e trecătoare.
  • Schimbarea clasei dependente poate afecta clasa utilizatoare.
  • Exemplu: un Mecanic folosește o Cheie ca să repare mașina, dar nu o deține permanent.
Mecanic
+ Repara(cheie: Cheie)
- - - -▶ dependență
Cheie
+ Tip : string
+ Strange()
C# 
public class Cheie
{
    public string Tip { get; set; }

    public void Strange() =>
        Console.WriteLine($"Cheie [{Tip}] strânsă.");
}

public class Mecanic
{
    public string Nume { get; set; }

    // Dependență — Cheie e parametru, nu câmp
    public void Repara(Cheie cheie, string componenta)
    {
        Console.WriteLine($"{Nume} repară {componenta}...");
        cheie.Strange();  // folosire temporară
        Console.WriteLine("Reparație finalizată!");
    }               // cheie iese din scope — Mecanicul nu o păstrează
}

Mecanic ion   = new Mecanic { Nume = "Ion" };
Cheie   cheie = new Cheie   { Tip  = "17mm" };

ion.Repara(cheie, "filtru ulei");
// Ion repară filtru ulei...
// Cheie [17mm] strânsă.
// Reparație finalizată!
